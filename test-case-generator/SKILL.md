# 测试用例生成器

本技能用于从需求文档生成标准化的 Markdown 格式测试用例，**核心增强功能是自动识别跨文档的重复功能点，将通用测试用例提取到复用文件，避免重复编写**。支持转换为 Excel 和思维导图可视化。

## 核心特性

### 1. 功能点智能去重

- 自动识别跨多个需求文档的相同/相似功能点
- 使用关键词匹配 + 语义分析判断功能点相似度
- 生成功能点分组报告供人工审核

### 2. 通用/特有用例分离

- **通用场景**: 跨模块的相同功能点测试用例 → `case_MD/0_用例复用.md`
- **特有场景**: 单模块独有功能点测试用例 → `case_MD/<模块名>_测试用例.md`

### 3. 批量生成优化

- 单次遍历所有 SRS 文档，提取全部功能点
- 一次性完成功能点去重和分组
- 分批次生成测试用例，避免重复

## 使用场景

- 用户提供单个需求文档并要求生成测试用例
- 用户提供多个需求文档并要求**批量生成测试用例**
- 用户需要为功能需求创建结构化测试用例
- 用户希望**避免测试用例重复**,提高复用性
- 用户希望生成可转换为 Excel 格式的测试用例
- 用户需要兼容 markmap 思维导图可视化的测试用例

## 文件目录结构

项目使用以下目录结构:

```
markdownToCase/
├── SRS/                           # 需求文档 (包含功能点分析)
│   ├── 1_唯一码收货.md
│   ├── 2_唯一码组盘.md
│   └── ...
├── case_MD/                        # 测试用例 MD 文件
│   ├── 0_用例复用.md               # 通用测试用例 (跨模块复用)
│   ├── 1_唯一码收货_测试用例.md
│   ├── 2_唯一码组盘_测试用例.md
│   └── ...
├── case_excel/                     # 生成的 Excel 测试用例文件
│   ├── 0_用例复用.xlsx
│   ├── 1_唯一码收货_测试用例.xlsx
│   └── ...
└── config/                         # 配置文件目录
    └── feature_groups.json         # 功能点分组配置
```

## 工作流程

### 阶段一：功能点提取与去重 (批量生成时执行)

#### 步骤 1: 遍历所有 SRS 文档

读取 `SRS/` 目录下所有需求文档，提取以下信息:

```json
{
  "module": "模块名称",
  "file_path": "SRS 文件路径",
  "features": [
    {
      "name": "功能点名称",
      "description": "功能描述",
      "keywords": ["关键词 1", "关键词 2"],
      "has_test_cases": false
    }
  ]
}
```

#### 步骤 2: 功能点相似度分析

**规则遵循**: `reference/similarity_rule.md`

````


#### 步骤 3: 生成功能点分组

将功能点分为两类:

**通用功能点**(跨 2 个及以上模块):
```json
{
"group_id": "FG001",
"feature_name": "唯一码验证",
"modules": ["唯一码收货", "唯一码组盘", "唯一码盘点"],
"commonality": 0.85,
"common_aspects": ["空唯一码校验", "格式校验", "特殊字符处理"],
"module_specific": {
  "唯一码收货": ["状态必须为初始化", "不能绑定容器"],
  "唯一码组盘": ["容器未锁定", "未开始组盘"]
}
}
````

**特有功能点**(仅单模块存在):

```json
{
  "group_id": "FG002",
  "feature_name": "收货完成容器绑定",
  "modules": ["唯一码收货"],
  "commonality": 0.0,
  "description": "收货完成后绑定到收货区库存容器"
}
```

#### 步骤 4: 保存功能点分组配置

将分组结果保存到 `config/feature_groups.json`,格式遵循: `references/group_rule.md`

### 阶段二：测试用例生成

#### 步骤 5: 生成通用测试用例

**目标文件**: `case_MD/0_用例复用.md`

**生成规则**:

1. 遍历所有 `type: "common"` 的功能点组
2. 为每个功能点组生成测试用例，覆盖:
   - **共性场景**: 所有模块通用的测试场景
   - **参数化设计**: 使用 `{模块名}` 占位符，便于理解适用范围

**示例**:

```markdown
# 通用测试用例

## 唯一码验证 (跨模块复用)

**适用模块**: 唯一码收货、唯一码组盘、唯一码盘点

### 空唯一码校验

- [P1] 用户已登录 PDA 系统;已进入对应模块页面 | 输入空唯一码;调用验证接口 | 接口返回错误提示;页面显示"唯一码不能为空"

### 唯一码格式错误校验

- [P1] 用户已登录 PDA 系统;已进入对应模块页面 | 输入超长唯一码 (超过系统限制) | 接口返回错误提示;页面显示"唯一码格式错误"

### 特殊字符唯一码校验

- [P2] 用户已登录 PDA 系统;已进入对应模块页面 | 输入包含特殊字符的唯一码 | 接口返回错误提示;页面显示"唯一码格式错误"
```

#### 步骤 6: 生成特有测试用例

**目标文件**: `case_MD/<模块名>_测试用例.md`

**生成规则**:

1. 为每个模块生成独立的测试用例文件
2. 包含两类测试用例:
   - **特有功能点测试用例**: 仅该模块独有的功能点
   - **通用功能点的模块特定场景**: 通用功能点中该模块特有的测试场景

**示例**(唯一码收货特有场景):

```markdown
# 唯一码收货

## 收货处理 - 容器绑定

### 收货完成绑定容器

- [P0] 用户已提交收货;所有唯一码收货成功 | 调用后端收货完成逻辑;绑定收货区库存容器 | 唯一码绑定到容器 CONT001;库存更新;页面显示收货成功

## 唯一码验证 - 收货特定规则

### 唯一码状态不为初始化

- [P1] 用户已扫码;解析成功;唯一码状态为已收货 | 调用后端唯一码验证接口 | 验证不通过;返回错误提示;页面显示"唯一码已收货，无法再次收货"
```

#### 步骤 7: 建立引用关系

在模块测试用例文件中添加引用说明:

```markdown
> **注意**: 本模块复用了以下通用测试用例，请在执行测试时参考 `0_用例复用.md`
>
> - 唯一码验证 (空唯一码、格式错误、特殊字符)
> - 扫码解析处理 (无效扫码、接口失败)
> - 网络异常处理
> - 登录与权限验证
```

### 阶段三：质量检查与优化

#### 步骤 8: 重复用例检测

**检测规则**:

1. 遍历所有生成的测试用例
2. 提取测试用例的关键特征:
   - 功能点名称
   - 前置条件关键词
   - 操作步骤关键词
   - 预期结果关键词

3. 计算用例相似度:

   ```
   similarity = (功能点匹配 + 步骤匹配 + 结果匹配) / 3
   ```

4. 标记重复用例:
   - `similarity >= 0.8`: 高度重复，建议合并
   - `0.6 <= similarity < 0.8`: 中度重复，人工审核

**输出报告**:

```markdown
## 重复用例检测报告

### 高度重复用例 (建议合并)

1. **唯一码格式校验**
   - 出现在：`0_用例复用.md`、`1_唯一码收货_测试用例.md`
   - 相似度：0.92
   - 建议：保留在 `0_用例复用.md`,从模块文件删除

2. **空唯一码校验**
   - 出现在：`0_用例复用.md`、`2_唯一码组盘_测试用例.md`
   - 相似度：0.88
   - 建议：保留在 `0_用例复用.md`,从模块文件删除
```

#### 步骤 9: 生成统计报告

```markdown
## 测试用例生成统计

### 总体统计

- 总功能点数：45
- 通用功能点：12(26.7%)
- 特有功能点：33(73.3%)
- 总测试用例数：156

### 用例分布

- 通用测试用例 (0\_用例复用.md): 68 条 (43.6%)
- 模块特有测试用例：88 条 (56.4%)

### 优先级分布

- P0(高): 42 条 (26.9%)
- P1(中): 67 条 (42.9%)
- P2(低): 35 条 (22.4%)
- P3(复用): 12 条 (7.7%)

### 去重效果

- 识别重复功能点：12 组
- 避免重复用例：约 45 条
- 用例精简率：28.8%
```

### 阶段四：输出与转换

#### 步骤 10: 保存测试用例

**保存规则**:

1. **通用测试用例**:
   - 文件：`case_MD/0_用例复用.md`
   - 格式：遵循 `references/case_template.md`
   - 标记：在文件顶部添加说明"本文件包含跨模块复用的通用测试用例"

2. **模块特有测试用例**:
   - 文件：`case_MD/<模块名>_测试用例.md`
   - 格式：遵循 `references/case_template.md`
   - 引用：在文件顶部添加对 `0_用例复用.md` 的引用说明

#### 步骤 11: Excel 转换 (可选)

仅在用户明确要求 `Excel 转换/用例表格转换` 时才执行

**两种转换模式**:

1. **单文件转换** - 导出单个模块的测试用例

   ```bash
   # 转换单个模块
   python scripts/md_to_excel.py case_MD/1_唯一码收货_测试用例.md
   python scripts/md_to_excel.py case_MD/0_用例复用.md

   # 指定输出路径
   python scripts/md_to_excel.py case_MD/1_唯一码收货_测试用例.md case_excel/唯一码收货.xlsx

   # 自定义用例编号前缀
   python scripts/md_to_excel.py case_MD/1_唯一码收货_测试用例.md --prefix TEST
   ```

   **适用场景**:
   - 需要单独分发某个模块的测试用例
   - 模块独立测试执行
   - 快速导出单个文件查看

2. **批量转换** - 合并所有模块到一个 Excel 文件

   ```bash
   # 批量转换所有文件
   python scripts/batch_md_to_excel.py

   # 指定输出路径
   python scripts/batch_md_to_excel.py case_excel/所有测试用例.xlsx

   # 自定义用例编号前缀
   python scripts/batch_md_to_excel.py --prefix TEST
   ```

   **适用场景**:
   - 生成整体测试用例报告
   - 跨模块测试用例审查
   - 统一管理和分发

**输出内容**:

- 从 `case_MD/` 目录读取测试用例文件
- 生成 Excel 文件到 `case_excel/` 目录
- Excel 文件包含以下列:
  - 用例编号 (格式：`DH_<模块缩写>_<序号>`)
  - 所属模块
  - 功能点
  - 优先级 (P0/P1/P2/P3，带颜色标识)
  - 用例标题
  - 前置条件 (带序号)
  - 操作步骤 (带序号)
  - 预期结果 (带序号)
  - 来源文件 (批量转换时包含)

#### 步骤 12: 思维导图可视化 (可选)

Markdown 格式兼容 markmap，可以将测试用例结构可视化为思维导图。

**命令**:

```bash
npx markmap case_MD/0_用例复用.md
npx markmap case_MD/1_唯一码收货_测试用例.md
```
