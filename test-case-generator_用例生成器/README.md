# 测试用例生成器

从需求文档自动生成标准化测试用例的工具，支持Markdown格式输出、Excel转换和思维导图可视化。

---

## 📋 功能概述

本工具帮助测试人员和开发人员快速从需求文档生成结构化的测试用例，主要功能包括：

- ✅ 从Markdown格式的需求文档自动提取功能点
- ✅ 生成符合规范的测试用例（Markdown格式）
- ✅ 支持测试用例复用（自动识别通用场景）
- ✅ 可转换为Excel格式便于管理和评审
- ✅ 兼容markmap思维导图可视化
- ✅ 自动分配优先级（P0/P1/P2/P3）

## 🎯 使用方法

### 方法一：通过AI技能调用（推荐）

本工具以AI技能的形式集成，当需要生成测试用例时，直接提供需求文档并要求"生成测试用例"即可。

### 方法二：手动执行脚本

#### 1. 创建必要文件夹

```bash
python scripts/create_folder.py folder "case_MD"
python scripts/create_folder.py folder "case_excel"
```

#### 2. 生成测试用例

AI技能会自动读取需求文档并生成Markdown格式的测试用例。

#### 3. 转换为Excel（可选）

```bash
python scripts/md_to_excel.py case_MD/模块名称_测试用例.md
```

或指定输出路径：

```bash
python scripts/md_to_excel.py case_MD/模块名称_测试用例.md case_excel/模块名称_测试用例.xlsx
```

## 📝 操作步骤

### 步骤1：准备需求文档

将需求文档（Markdown格式）放在 `SRS/` 目录下，例如：

```
SRS/
├── 1. 唯一码收货.md
└── 2. 唯一码组盘.md
```

### 步骤2：检查通用测试用例

工具会自动检查 `case_MD/_通用测试用例.md` 是否存在，如不存在会自动创建。

### 步骤3：读取模块配置

工具会自动检查 `config/module_config.json` 是否存在，如不存在会自动创建。

### 步骤4：生成测试用例

提供需求文档并要求"生成测试用例"，AI将自动：

1. 分析需求文档中的功能点
2. 插入功能点分析段落到源文档
3. 识别可复用的通用场景
4. 生成结构化测试用例
5. 保存到 `case_MD/` 目录

### 步骤5：转换为Excel（可选）

如需生成Excel格式，执行：

```bash
python scripts/md_to_excel.py case_MD/模块名称_测试用例.md
```

## 📊 测试用例格式

### Markdown格式

```markdown
# 模块名称

## 功能模块名称

### 测试用例标题

- [Px] 前置条件 | 操作步骤 | 预期结果
```

### 格式说明

| 符号   | 说明         | 示例               |
| ------ | ------------ | ------------------ | ------------------------ |
| `#`    | 模块名称     | `# 唯一码收货`     |
| `##`   | 功能模块     | `## 扫码处理-解析` |
| `###`  | 测试用例标题 | `### 扫码解析成功` |
| `[Px]` | 优先级       | `[P0]`、`[P1]`     |
| `      | `            | 字段分隔符         | 前置条件 \| 步骤 \| 预期 |
| `;`    | 多条件分隔   | 条件1;条件2        |

### 优先级定义

| 优先级 | 含义 | 使用场景                         |
| ------ | ---- | -------------------------------- |
| **P0** | 高   | 核心功能、主流程、关键业务场景   |
| **P1** | 中   | 异常流程、错误处理、重要边界场景 |
| **P2** | 低   | 边界测试、极端情况、次要场景     |
| **P3** | 最低 | 共享/复用型测试用例              |

### 示例

```markdown
# 唯一码收货

## 扫码处理-解析

### 扫码解析成功并填充字段

- [P0] 用户已登录PDA系统;已进入收货页面;存在未扫描的唯一码 | 扫描有效的唯一码;观察页面字段变化 | 调用后端唯一码解析接口成功;解析结果填充到对应字段;页面显示解析后的数据

### 扫码解析无返回不做处理

- [P1] 用户已登录PDA系统;已进入收货页面 | 扫描无效唯一码;观察页面变化 | 调用后端唯一码解析接口;接口无返回数据;页面保持原状，不做任何填充处理
```

## 🔧 模块配置

模块配置文件 `config/module_config.json` 用于定义模块的通用场景和特有场景：

```json
{
  "modules": {
    "模块名称": {
      "abbr": "模块缩写",
      "action": "动作",
      "page": "页面名称",
      "api_name": "接口名称",
      "common_scenarios": ["通用场景1", "通用场景2"],
      "unique_scenarios": [
        {
          "name": "特有场景名称",
          "priority": "P1",
          "preconditions": "前置条件",
          "steps": "操作步骤",
          "expected": "预期结果"
        }
      ]
    }
  }
}
```

## 📈 Excel输出

### 表格结构

| 列名     | 说明                      |
| -------- | ------------------------- |
| 用例编号 | 自动生成的唯一编号        |
| 所属模块 | 测试用例所属模块          |
| 功能点   | 对应的功能模块            |
| 优先级   | P0/P1/P2/P3（带颜色标识） |
| 用例标题 | 测试用例名称              |
| 前置条件 | 执行用例前的准备条件      |
| 操作步骤 | 执行步骤（自动添加序号）  |
| 预期结果 | 预期结果（自动添加序号）  |

### 优先级颜色

- **P0（高）**：红色背景
- **P1（中）**：黄色背景
- **P2（低）**：绿色背景
- **P3（共享）**：灰色背景

### Excel示例命令

```bash
# 使用默认输出路径
python scripts/md_to_excel.py case_MD/唯一码收货_测试用例.md

# 指定输出路径
python scripts/md_to_excel.py case_MD/唯一码收货_测试用例.md case_excel/测试用例.xlsx

# 自定义编号前缀
python scripts/md_to_excel.py case_MD/唯一码收货_测试用例.md --prefix TEST
```

## ⚠️ 注意事项

### 文件命名规范

- 需求文档：使用清晰的名称，如 `1. 唯一码收货.md`
- 测试用例：使用 `模块名称_测试用例.md` 格式
- 避免使用中文标点符号和特殊字符

### 优先级分配原则

- **P0**：每个功能至少3-5个核心场景
- **P1**：覆盖主要异常和边界情况
- **P2**：补充极端情况和细节测试
- **P3**：复用通用场景（如登录、导航等）

### 格式检查清单

- [ ] 模块名称使用 `#` 标记
- [ ] 功能模块使用 `##` 标记
- [ ] 用例标题使用 `###` 标记
- [ ] 每个用例以 `- [Px]` 开头
- [ ] 字段之间使用 `|` 分隔
- [ ] 多条件使用 `;` 分隔
